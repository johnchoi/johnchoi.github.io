1. 概述

1.1 名词说明

名称 说明
摘要(Abstract) 摘要通常在正式报告前几个月就要写好，摘要可以帮助作者取舍报告的内容。主办方（评审委员）通过摘要评审来决定是否需要选择该作者作为会议的演讲者、或者在摘要集中为作者的工作进行宣传和吸引人们来参加作者的讨论或观看电子壁报。因此，一般来说，摘要应该有坚实且令人兴奋的结果，但还要给人说明作者从事的研究工作是动态的、仍在进行的，读过摘要的人只有通过听取报告才能够明白你研究的内容。在学术会议中，摘要的第一位读者就是会议论文的评审委员会
通讯作者(Corresponding Author) 通讯作者往往是课题的总负责人，承担课题的经费，设计，文章的书写和把关。也是文章/摘要和研究材料的联系人。最重要的是，他担负着文章可靠性的责任。需要负责与主办方（评审委员会）的一切通信联系和接受读者的咨询等。实际上如果从知识产权上来说，研究成果算是通讯作者的。通讯作者的好处是能和外界建立更广泛的联系。通讯作者通常由论文所属领域内最有权威者担当。
第一作者(First Author) 必须是第一线操作的实施者和原始数据的收集和处理人，又是初稿的执笔人，第一作者要对研究结果和数据真实性负首要责任，第一作者的署名通常排在最前面。
联合作者(Co-Author) 对作品有一定直接或间接帮助的贡献人，在总体贡献上跟第一作者有较大差距，统称为联合作者
主办方(Organizer) 学术会议的发起人
大会主席(Conference chair) 以公正的立场围绕会议主题，主持并引导协调会议的人
评审委员会(Selection Committee) 由评审专家组成的摘要评审组织
评审专家(Reviewer) 对摘要进行评审的某领域的专家
评审任务(Review quest) 由评审委员会或主办方将摘要分配给评审专家而产生的任务

1.2 产品概述及目的

1.2.1 产品概述

征文摘要管理系统主要目的是为了给学术会议接受和准备演讲摘要使用，通常在会议开始前几个月，会议主办方或评审委员会需要收集学者在最近一段时间内的工作成果摘要或说明。摘要基本上说明了某种假设，研究使用的工具，收集的数据以及对这些数据的基本理解。

大会主席或评审委员会可以选择接受或拒绝这些摘要，被接受的摘要将会被安排到会议日程中，作为口头演讲或者电子壁报。摘要通常也会在会议之前或之后，被整理成会议记录、学术日志、摘要集或在线发布（目前尚未处理）。

1.2.2 产品目的

征文摘要管理系统包含了摘要提交设置、摘要提交、摘要管理以及摘要评审等四大部分，操作人涉及到后台管理员、作者以及评审专家等多种不同的身份。

征文摘要管理系统使得这些学术会议中的三种不同身份的角色能够使用系统进行无缝合作，使学术会议主办方在收集、管理、评审摘要时更加高效快捷。

第一期产品的目的包括：收集摘要、分配评审任务、获取最终评审结果。

第一期产品不包含的内容有：将摘要制作成摘要集、会议记录或学术日志。

1.3 产品风险

风险级别说明：非常高的，高的，适度的，低的，非常低的

风险 风险级别 描述 监控策略 改善策略
与原有报名流程有一定的区别 适度的 学术会议的注册报名流程与原有会议的报名流程有所不同，一般来说需要参会人先注册后报名，参会人提交某一部分信息后有可能经过很长一段时间后才会报名付款或提交征文，所以需要改造目前的报名流程，可能会存在一定的流程无法完全适配的情形 需要通过服务一两场学术会议来了解具体的流程 跟踪获得数据后，需要根据实际情况不断的调整，使之更适合学术会议
对于学术会议不够了解，可能导致系统不完全满足某些学术会议的需求 高的 由于产品及开发都对学术会议不够了解，对于某些细节，只能通过从竞品或网上的资料获得的信息进行主观判断，可能会导致某些需求理解上有偏差，无法完全满足需求 需要通过服务一两场学术会议来了解具体的流程 需要获得真实会议反馈后，重新理解需求

2.2.1.2 启用摘要提交
  • 简要说明

启用摘要提交后，参会人/作者登录个人中心后，才能够看到提交摘要菜单

  • 业务规则

  1. 如果未启用摘要提交，则参会人/作者在个人中心中无法看到提交摘要的菜单
  2. 如果已启用摘要提交，则参会人/作者在个人中心中可以看到提交摘要的菜单
  3. "启用摘要提交"功能属于专业版功能，需要开通专业版后方能使用
  4. 如果当前会议不是专业版，则无法开启“启用学术会议”，也就无法看到“启用摘要提交”设置项，但是在“启用摘要提交”的地方需要再做一次判断，确保逻辑正确
2.2.1.3 摘要提交时间范围
  • 简要说明

设置摘要提交的开始和结束时间，在当前时间超出该时间段（之前或之后）时，无法创建或修改摘要提交，需修改状态下的摘要重新提交不受该时间段限制。

  • 业务规则

  1. 征文提交开始时间：任何时间皆可；结束时间：晚于开始时间；开始和结束时间为必填项，不能为空
  2. 当前时间在摘要提交时间范围之前，提示“摘要提交尚未开始，摘要提交时间：2017-04-02 09:12 至 2015-05-02 09:12”，见图3
  3. 当前时间在摘要提交时间范围之后，提示“摘要提交已截止，无法继续提交”，见图4
  4. 已提交过摘要的作者，如果当前时间在提交时间范围之后，在摘要显示列表中，禁用重新提交按钮，鼠标经过时提示“摘要提交已截止，无法重新提交”，见图5
  5. 当摘要已被评审专家或最终评审标记为“需修改”状态时，不在该时间范围内也能够重新提交
  6. 默认值为空

2.2.1.8 学科分类
  • 简要说明

设置作者在提交摘要时可选择的学科分类，作者在提交摘要时，必须选择其中的一个分类作为主题。我们会预设一整套学科分类，数据将从教育部数据库中获取。主办方可从预设的数据库中添加分类或自定义。

  • 业务规则

  1. 左侧穿梭框内为我们提交预设好的学科分类数据，最多为3级结构，分为中英文双语，且每个学科分类带有分类标识，用于生成征文序号
  2. 左侧框内可以进行学科分类搜索，搜索时会显示包含该关键字的所有层级，如第一级包含“关键字”，则显示该级下所有分类；如第二级包含“关键字”，则显示其上一级分类、包含关键字本身的二级分类以及其下所有三级分类；如第三级包含“关键字”，则显示其本身以及二级和一级分类。
  3. 左侧框内的数据无法编辑

2.2.6 征文提醒设置

  • 简要说明

在征文状态变更时会使用短信或邮件向作者或评审专家进行提醒

  • 业务规则

发送给主办方的通知：

1.任务接受提醒：

  • 触发条件：在摘要评审任务被评审专家接受时发送给主办方管理员
  • 内容：征文编号为：ABC-123的评审任务已被评审专家{专家姓名}接受，请登录后台查看；
  • 默认状态：不勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、专家姓名、专家手机、专家邮箱、摘要编号、任务ID、任务接受截止时间、征文学科分类、摘要标题、任务接受时间

2.任务拒绝提醒：

  • 触发条件：在摘要评审任务被评审专家拒绝时发送给主办方管理员
  • 内容：征文编号为：ABC-123的评审任务已被评审专家{专家姓名}拒绝，请登录后台查看
  • 默认状态：不勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、专家姓名、专家手机、专家邮箱、摘要编号、任务ID、任务接受截止时间、征文学科分类、摘要标题、任务拒绝时间、拒绝原因

3.评审邀请退回提醒：

  • 触发条件：在摘要评审任务到截止时间后未被接受或拒绝，被退回时发送给主办方管理员
  • 内容：征文编号为：ABC-123的评审任务已到截止时间，评审专家{专家姓名}未接受该任务，已被退回，请登录后台查看
  • 默认状态：不勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、专家姓名、专家手机、专家邮箱、摘要编号、任务ID、任务接受截止时间、征文学科分类、摘要标题、任务退回时间

4.评审任务退回提醒：

  • 触发条件：在摘要评审任务到截止时间后未被评审，被退回时发送给主办方管理员
  • 内容：征文编号为：ABC-123的评审任务已到截止时间，评审专家{专家姓名}未进行评审，已被退回，请登录后台查看
  • 默认状态：不勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、专家姓名、专家手机、专家邮箱、摘要编号、任务ID、任务接受截止时间、征文学科分类、摘要标题、任务退回时间

5.评审任务完成提醒：

  • 触发条件：当评审任务被评审专家评审完成后，发送给主办方管理员
  • 内容：编号为：ABC-123的摘要已评审完成，评审结果为“{评审结果}”，请登录后台查看
  • 默认状态：不勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、专家姓名、专家手机、专家邮箱、摘要编号、任务ID、任务接受截止时间、征文学科分类、摘要标题、任务完成时间

发送给作者的提醒：

1.提交成功提醒：

  • 触发条件：在征文被作者提交成功时发送给作者
  • 内容：您的摘要已提交成功，在{截止时间}之前，您可以使用手机号码登录后台修改征文内容
  • 默认状态：勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、学科分类、个人中心URL、征文标题、提交时间

2.摘要修改提醒：

  • 触发条件：当摘要被评审专家或主办方标记为“需修改”时，发送给作者
  • 内容：您在[会议名称]中提交的摘要评审结果为：“小幅修改”，请登录个人中心查看
  • 默认状态：勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、学科分类、个人中心URL、征文标题、提交时间、征文状态

3.评审结果提醒：

  • 触发条件：a. 当评审任务被评审专家评审完，且不需要最终评审时；b.需要最终评审，并且主办方在后台进行了最终评审时；发送给作者的通知
  • 内容：您提交的摘要已评审完成，评审结果为：{评审结果}，请登录个人中心查看评审结果
  • 默认状态：勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、学科分类、个人中心URL、征文标题、提交时间、评审结果

发送给评审专家的提醒：

1.分配评审提醒：

  • 触发条件：在摘要被分配给评审专家时发送评审专家 ## > * 内容：您已收到一条新的评审邀请，征文编号：ABC-123，请登录查看 {loginUrl}
  • 默认状态：勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、学科分类、个人中心URL、征文标题、提交时间、任务接受截止时间、任务分配时间

2.摘要已修改提醒：

  • 触发条件：当摘要被作者修改完成后，再次提交时，发送给评审专家
  • 内容：编号为：ABC-123的摘要已修改完成，请登录后台查看
  • 默认状态：勾选
  • 可插入字段：作者姓名、作者手机、作者邮箱、学科分类、个人中心URL、征文标题、提交时间、任务评审截止时间、任务分配时间

3. 常见问题

3.1 问题一：勾选行中包括了有任务的专家（备注：已经被分配任务的专家无法删除）以及没有任务的专家，那么点击批量删除后，结果如何？

相关问题：专家管理这里 批量删除的时候 已经有任务的专家应该是不能删除的吧 这里是不是要把有任务的专家前面的复选框变成不可选的状态呢？

答：考虑到以后可能会添加其他批量操作的情况，需要允许选中已有任务的专家；当选中的专家中即有没有任务的专家，也有已有任务的专家时，禁用“批量删除”操作，鼠标经过时提示：您选择的专家中已被安排评审任务，无法批量删除

3.2 问题二：征文列表中的“已修改”是按钮还是标签？

答：“已修改”是一个标签，表示当前征文摘要已经被作者修改过了

3.3 问题三：征文列表中每个选项卡中对应的操作具体是如何对应的？比如点击已评审，是否只有最终评审和归档呢？

相关问题一：分配评审也会存在补充图中吗？则该图中的单篇评审图标与分配评审有何区别？

相关问题二：点击上方的每一个选项卡， 列表中的 操作列如何一一对应的？（比如此图中 操作列3个操作）

答：

  • “全部”列表中，存在“归档”、“删除”两种操作
  • "待分配"列表中，存在“归档”、“删除”、“分配评审”、“最终评审”四种操作
  • “待接受”列表中，存在“归档”、“删除”、“取消评审任务”三种操作
  • “待评审”列表中，存在“归档”、“删除”、“取消评审任务”三种操作
  • “已评审”列表中，存在“归档”、“删除”、“最终评审”三种操作
  • “已最终评审”列表明中，存在“归档”、“删除”、“修改评审”三种操作

注意：”取消评审“任务操作会将评审任务取消，专家在摘要列表里将无法看到该任务和征文摘要，需要二次确认，提示文案：取消评审将会使该任务所属评审专家无法再看到该任务
“修改评审”可以修改已经存在的评审结果，使用通用的评审模态窗口

相关问题一回答：分配评审只存在与“待分配”列表中，与单篇评审图标的分配评审没有区别，只是一个是批量，一个是单个

相关问题二回答：在“全部”“待分配”中，存在“编辑”、“删除”、“分配评审”三个按钮，在其他列表中，仅有“编辑”、“删除”按钮
